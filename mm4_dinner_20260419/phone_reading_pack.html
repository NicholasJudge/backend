MM4 Wireframe

This note is the compact reference for Robert_MM_4 as built on 2026-04-18. It describes the current model stack, the feature flow between layers, the statistical learning contract, and the control-board surfaces needed to diagnose failure modes.

MM4 preserves the Robert outer-loop structure, but retargets that loop to the final live object. The notebook, control board, frozen eval window, publish path, generated live config, and launch or smoke commands are all MM4-scoped and judged against the joint execution transformer used by the real live runner.

4.2 Supervision

The loss is a weighted sum of nine heads:

head target type loss current weight
fill binary BCE 0.08
adverse binary BCE 0.35
edge short continuous SmoothL1 0.08
edge long continuous SmoothL1 0.12
value continuous SmoothL1 1.00
alpha short continuous SmoothL1 0.02
alpha long continuous SmoothL1 0.04
inventory residual continuous SmoothL1 0.06
inventory gate continuous SmoothL1 0.06

Target construction:

  • value is the main ranking target. When available, it comes from offline unit replay: passive entry, pathwise inventory marking, stale-inventory requoting, and a stale-inventory penalty.
  • inventory_residual is a bounded transform of alpha_long.
  • inventory_gate is a bounded magnitude transform of alpha_long.

4.3 Why it is only RL-like

The RL-like component is indirect:

  1. replay converts a candidate action and the subsequent path into economic consequences;
  2. those consequences supervise value, fill, adverse, and inventory-control heads jointly;
  3. the learned inventory heads alter the next planner output;
  4. held-out model selection is based on replay economics and behavioral constraints, not only prediction loss.

What is absent:

  • no policy-gradient update;
  • no Bellman backup;
  • no online exploration term;
  • no direct live RL loop in the current MM4 baseline.

5. Live Decision Path

At each live decision time:

  1. build forecast_features = fcst_* + m1_*;
  2. build inventory and quality features from the ledger and streams;
  3. run the baseline planner with zero micro signal;
  4. run MM4 inventory heads with market state + forecast block + baseline planner state;
  5. rerun the planner with the inferred micro signal;
  6. generate JR_exec candidates;
  7. score candidates with MM4 candidate heads;
  8. select the candidate using predicted edge, fill, adverse risk, inventory gate, and a strong near-touch preference;
  9. log the realized or replay consequence for the next training pass.

This is the main information-flow test. If useful information exists upstream but does not change the final candidate ranking, the failure can occur in forecast aggregation, planner transmission, inventory heads, candidate heads, or the final selection rule.

Representation Learning Report

From representational_learning_mm4_report.md.

World-Model Representation Learning for Robert_MM_4

Scope and Thesis

The right object of study here is not market microstructure in isolation. It is representation learning for partially observed sequential control, with market making as the application domain. The central question is whether Robert_MM_4 should continue to rely primarily on hand-constructed features, or instead learn a high-dimensional latent state from a relatively small number of observed columns and event streams. My conclusion is that the latter is the scientifically stronger direction.

More concretely, the relevant problem is to learn a map

[ x_t \in \mathbb{R}^{d} \longmapsto z_t \in \mathbb{R}^{m}, \quad m \gg d, ]

where (x_t) is a compact observed state and (z_t) is a learned latent representation that is useful for predicting future latent states, future observations, and control-relevant quantities under actions. This is a world-model or predictive-state viewpoint. It is also naturally semi-supervised: unlabeled sequential data are abundant, whereas labels such as realized fill quality, adverse selection, and value are sparse and delayed.

In that sense, the correct comparison is not "engineered features versus a bigger supervised model." It is "engineered static summaries versus self-supervised predictive representations with supervised readouts."

Part I. General Representation Learning: The Relevant Literature

1. The core scientific shift: from descriptive features to predictive state

The older feature-engineering paradigm assumes that a human should specify the small set of statistics that matter. The modern representation-learning paradigm assumes instead that the system should discover latent variables that preserve what is useful for prediction and control while discarding nuisance variation. In sequential problems, this usually means that a good representation must satisfy three conditions:

  1. it must be predictive of the future;
  2. it must be invariant to nuisance details that do not affect future outcomes;
  3. it must remain usable under action-conditioning, because control changes the future.

This is why the strongest recent work has moved away from hand-crafted features and, increasingly, away from raw-space reconstruction. The best results come from learning in latent space.

2. Predictive latent objectives, not observation-space reconstruction, are the main line of progress

Contrastive Predictive Coding (CPC) is one of the clearest early formulations of this view. Van den Oord, Li, and Vinyals propose learning useful representations by predicting the future in latent space with an autoregressive model, using a contrastive objective rather than supervised labels [1]. The key scientific point is that the representation is trained to preserve information that is maximally useful for future prediction, not to preserve everything in the observation.

BYOL established that strong representations can be learned without negative pairs by training one network to predict another network's latent representation under a different view of the same input [2]. This matters because it showed that useful representations do not require a contrastive dictionary of negatives; consistency and asymmetry can suffice.

MAE pushed masked modeling into a scalable regime by reconstructing masked content from partial context [3]. However, it also clarified an important limitation: when the target is raw pixels, the decoder can dominate the task and the learned latent may still expend capacity on low-level detail. This is precisely the concern that motivates the move toward representation-space prediction.

Data2vec is especially relevant because it generalizes the masked latent-prediction idea across speech, vision, and language [4]. Its core move is to predict latent representations of the full input from a masked view in a self-distillation setup. This is conceptually close to what we would want in a general sequential system: the same predictive objective should work across modalities and should not depend on a bespoke task-specific label design.

3. LeCun's world-model program is the right conceptual frame, but the evidence comes from concrete JEPA systems

Yann LeCun's "A Path Towards Autonomous Machine Intelligence" is a position paper, not an empirical benchmark paper, so it should not be treated as evidence by itself [5]. Its value is that it states the architectural claim cleanly: intelligent systems need world models learned largely in a self-supervised manner; these models should support prediction, reasoning, and planning; and they should predict in a latent representation space rather than in raw observation space whenever possible.

That program is made concrete by I-JEPA. Assran et al. introduce a non-generative self-supervised architecture that predicts the latent representations of target blocks from a context block, explicitly aiming for semantic rather than low-level representations [6]. This is the closest direct empirical realization of the LeCun position that is mature enough to take seriously. The important lesson is not "predict missing image patches." The lesson is: define the self-supervised target in representation space, choose masking so that the model must learn semantic structure, and avoid forcing the system to spend capacity on irrelevant pixel detail.

For our purposes, JEPA should be understood as a family of latent predictive objectives. The key idea is to learn a state that is predictive of the unknown or future part of the world at the level of abstract structure, not sensory minutiae.

4. Action-conditioned world models are the bridge from representation learning to control

Once actions matter, one needs an action-conditioned latent dynamics model. PlaNet is a canonical reference here. Hafner et al. learn a latent dynamics model from images and plan in latent space, using deterministic and stochastic components and a multi-step objective ("latent overshooting") to improve long-horizon prediction [7]. The scientific point is that the latent state is judged by whether it supports accurate future reward-relevant dynamics, not by whether it can perfectly decode the input.

DreamerV3 extends this line and shows that robust world-model training can be scaled across many domains with a single configuration [8]. It remains an arXiv paper rather than a conference publication, so it should not bear the full argumentative load, but it is a serious and influential reference because it demonstrates a practical consequence of stable latent world models: once the dynamics are learned, policy improvement can happen partly through imagination in latent space.

There is also a more minimalist variant that is directly relevant to our use case. Self-Predictive Representations (SPR) learn an encoder whose latent state predicts future latent states under actions, while also enforcing consistency across augmented views [9]. This is close in spirit to what market making needs: good state representations should be predictive under a short-horizon action-conditioned transition model, even when labels are scarce.

Finally, the ICLR 2024 work "Bridging State and History Representations" argues that many successful RL representation-learning schemes can be understood through a common notion of self-predictive abstraction [10]. That is a useful unifying statement. It suggests that the correct mathematical object in sequential decision problems is not a bag of features, but a history representation that predicts its own future evolution and the future quantities relevant to control.

5. What this implies for "semi-supervised" learning

The strongest general lesson from the above literature is that semi-supervised learning in sequential domains should usually take the following form:

  1. use large streams of unlabeled data to learn a predictive latent state;
  2. use much smaller labeled datasets to fit task heads on top of that state;
  3. continue adapting the latent model slowly and the task heads faster as new data arrive.

This is exactly the regime of market making. Unlabeled data are effectively unlimited: quote streams, order-book snapshots, trades, cancellations, amendments, own decisions, own fills, and linked forecast logs. By contrast, clean labels for realized quote quality, adverse selection, or long-horizon inventory consequences are sparse, delayed, and expensive.

Therefore, the natural object of learning is not a fixed feature table. It is a latent world state trained first by self-supervision, then refined by sparse supervised targets.

6. Stability and continual adaptation

The user requested two specific issues: temporal stability of parameters and a smooth path from an offline-trained model to live online learning.

The relevant literature suggests that stability is not achieved by simple repeated dense SGD on the whole network. Some part of the system should be stable, and some part plastic. The cleanest way to achieve this is:

  1. pretrain a backbone world model offline;
  2. keep most of that backbone slowly changing or frozen;
  3. localize plasticity to a small subnetwork, such as adapters and heads;
  4. regularize that plastic subnetwork with anchoring and sparsity.

This is where lasso-like or (L_0)-like penalties make sense. They are most useful when applied to the plastic subnetwork, not indiscriminately to the whole backbone. The goal is not simply to make weights small. The goal is to maintain a compact, stable update locus so that the online learner can adapt without catastrophic drift.

A Short Finance-Specific Translation

The representation-learning literature above is general. Why should it apply to markets? Because markets are partially observed sequential systems with abundant unlabeled data, sparse action-conditioned labels, strong nuisance variation, and short-horizon control.

The finance literature gives supporting evidence but should not dominate the theory. Sirignano and Cont show that price formation has stable path-dependent structure across assets and time, which argues for learned sequence representations rather than single-step handcrafted summaries [11]. DeepLOB shows that structured architectures over order-book state can learn useful features automatically [12]. Kolm, Turiel, and Westray show that stationary order-flow-derived representations can outperform raw-book inputs, which is a reminder that the correct input space is the one that best preserves predictive structure, not necessarily the visually rawest one [13].

So the scientific thesis is: the correct market-making state is a learned predictive latent state, not a manually curated list of dozens or hundreds of summary features.

Part II. What This Means for Robert_MM_4

1. Current MM_4 should be read as a strong supervised readout on top of engineered state

The existing system is already more sophisticated than a shallow baseline. The core model is JointMicroTransformerV1 in nv/joint_mods/scorer.py, a multi-head transformer that ingests market, action, forecast, inventory, and candidate information. This is a reasonable readout architecture.

But from a representation-learning standpoint, the current stack is still dominated by engineered state compression. The local feature contract is broad in count but manual in semantics:

  1. market state in nv/joint_mods/market_state.py is compressed to 40 scalar features;
  2. recent own behavior in nv/joint_mods/execution_memory.py is compressed into short-window rates and means;
  3. forecast state in nv/joint_mods/forecasting.py and nv/joint_mods/context.py is expanded into many static m1_* and fcst_* columns;
  4. the frozen dataset manifest shows that the current training set is entirely replay_surface.

Thus the present system has a respectable supervised learner, but not yet a learned world representation in the stronger sense used in modern self-supervised learning.

2. The main recommendation is to learn a latent state from fewer primitive streams, not to hand-engineer still more columns

The most important change in viewpoint is this: MM_4 should likely move toward fewer hand-engineered columns, not more.

Right now, many derived channels such as EMA features, quality_* features, and numerous linked-signal expansions are attempts to manually manufacture a good state. A world-model approach would instead start from a smaller causal basis, for example:

  1. a compact market stream: top-of-book and near-book prices and sizes, trades, and event timing;
  2. own-action and own-outcome events: quote placement, amend, cancel, first-fill, terminal outcome;
  3. current inventory and simple account state;
  4. linked directional/world-model outputs as a short history, not only the latest row.

From these relatively few primitive streams, the model should learn a higher-dimensional latent state (z_t), for example 256-1024 dimensions, possibly overcomplete relative to the raw input dimension. The learned features in (z_t) are the substitute for feature engineering.

3. The right architecture for MM_4

For this repository, the most plausible architecture is:

  1. an encoder (e_\theta) that maps a recent history of market, action, forecast, and inventory observations to a latent state (z_t);
  2. an action-conditioned transition model (f_\theta) that predicts future latent states (z_{t+1}, \dots, z_{t+H}) under candidate actions;
  3. a small set of supervised heads (h_\phi) for fill probability, adverse selection, edge retention, value, and inventory residuals.

This preserves the parts of the current system that are already correct:

  1. explicit inventory remains explicit;
  2. explicit action candidates remain explicit;
  3. explicit downstream heads remain explicit.

What changes is the representation beneath them. The current transformer should be reinterpreted as a candidate readout layer, not as the entire learning story.

4. A scientifically coherent training objective

The objective for MM_4 should combine self-supervised prediction, action-conditioned latent dynamics, sparse supervised readouts, and stability terms. A reasonable template is

[ \mathcal{L}

\lambda_{\text{ssl}}\mathcal{L}{\text{masked-latent}} + \lambda{\text{dyn}}\mathcal{L}{\text{future-latent}} + \lambda{\text{obs}}\mathcal{L}{\text{selected-observables}} + \lambda{\text{sup}}\mathcal{L}{\text{fill/adverse/value}} + \lambda{\text{anchor}}\Omega_{\text{anchor}} + \lambda_{\text{sparse}}\Omega_{\text{sparse}}. ]

The terms mean:

  1. masked-latent: a JEPA/data2vec-style loss, where part of the history is hidden and the model predicts latent representations of the hidden part;
  2. future-latent: an action-conditioned SPR/PlaNet-style loss, where the model predicts future latent states under candidate actions;
  3. selected-observables: not full reconstruction, but prediction of a small set of control-relevant observables such as spread regime, queue depletion, short-horizon mid move, or forecast revision class;
  4. fill/adverse/value: the existing supervised market-making heads;
  5. anchor: penalty to a prior competent parameter state;
  6. sparse: lasso/group-lasso/(L_0)-style sparsity on the plastic subnetwork only.

This is much closer to LeCun's world-model framing than the current purely supervised feature table pipeline.

5. Why MM_4 is a good fit for this

This repository has exactly the asymmetry that favors semi-supervised representation learning:

  1. there is a large supply of unlabeled sequential data;
  2. there is a smaller supply of replay labels;
  3. live action-conditioned labels are sparse but economically important.

The current manifest reports 105,221 rows, all from replay_surface. That means the current supervised dataset is not yet the real live-action distribution. This is not a reason to avoid representation learning. It is a reason to use it: pretrain the latent state on the much larger unlabeled stream, then use replay labels only to fit the readout layer and initial calibration.

This also addresses one weakness visible in the current training logs: the latest evaluation snapshot reports essentially zero value correlation and negative replay PnL on the pinned evaluation window. That is exactly the empirical signature of a representation bottleneck. The readout exists, but the learned state is not yet sufficiently discriminative.

6. Response to the two requested practical issues

(A) Over-time stability: lasso plus zeroing

The repository already contains a primitive version of this idea through lambda_prev_l2 and prune_weakest_frac. Conceptually, that is pointing in the right direction, but it is too blunt if applied globally.

My recommendation is:

  1. keep a largely stable backbone world model;
  2. define a small plastic subnetwork, ideally adapters plus the output heads;
  3. apply lasso, group-lasso, or approximate (L_0) regularization only to this plastic subnetwork;
  4. if hard zeroing is used, zero the smallest weights only inside that plastic subnetwork;
  5. maintain an anchoring penalty to the previous competent state;
  6. replay offline data during online updates so the support does not drift rapidly.

So the answer is yes, but the object of shrinkage should be the plastic module, not the entire model.

(B) Smooth learning from offline training to real actions

The clean path is:

  1. Stage 0: offline self-supervised pretraining. Train the encoder and transition model on unlabeled market and event streams.
  2. Stage 1: offline supervised readout fitting. Fit the existing heads on replay-surface labels.
  3. Stage 2: online semi-supervised continuation. Continue self-supervised updates online on unlabeled live streams, but at a low rate and with anchoring.
  4. Stage 3: online supervised head/adaptor updates. Update only the plastic readout module on realized live action outcomes.
  5. Stage 4: rare backbone refreshes. Unfreeze a slightly larger subset only after enough live data accumulate.

This is smoother than periodic stop-and-retrain cycles because the backbone is already learning from unlabeled stream continuity, while the high-risk supervised updates are localized.

7. What I would and would not change in MM_4

I would change:

  1. the philosophy of state construction, from manual expansion toward predictive latent learning;
  2. the training schedule, from mostly supervised replay fitting toward self-supervised pretraining plus supervised readouts;
  3. the online update path, from a separate shallow refitter toward a shared latent backbone with small plastic online heads.

I would not change:

  1. the explicit inventory channel;
  2. the explicit action/candidate structure;
  3. the multi-head decomposition into fill, adverse selection, value, and inventory-related targets;
  4. the presence of a shallow linear sidecar, which is still useful as a calibration benchmark.

Bottom Line

The correct next move for Robert_MM_4 is not more market-specific feature engineering. It is general predictive representation learning applied to a market-making problem.

In the most compact form: learn a world model first, then attach market-making heads to it.

The representation should be learned from compact primitive streams and should expand them into a higher-dimensional latent state that is predictive, invariant to nuisance variation, and action-conditioned. That latent state should be trained mostly from self-supervision and only secondarily from sparse labels. Online learning should continue in that same latent space, with plasticity localized to a small sparse subnetwork and protected by anchoring and replay.

That is the LeCun/JEPA/world-model interpretation of what MM_4 should become.
